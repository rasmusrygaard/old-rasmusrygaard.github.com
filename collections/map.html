Here, we store our histogram in the Map<int, int> called histogram. This map stores mappings from keys to the number of occurrences for that key. Before we add each value, we check if we have seen that value before. If not, we add a new mapping to the value 1 since we have seen value exactly once. Otherwise, we increment this count.

This seems reasonable, but the CS106B libraries lets us write something even more convenient. Looking at the map documentation, we see that the [] operator has two cases:

- If ```value``` is a key in the map, ```histogram``` returns a reference to the ```int``` in the map.
- If ```value``` is *not* a key in the map, three things happen:
    1. We create a default value of type ValueType. In this case, we get an ```int``` with value 0.
    2. The ```map``` adds a mapping from ```values``` to this value.
    2. The ```map``` returns a reference to this new value.
